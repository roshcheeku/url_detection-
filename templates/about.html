Backend Workflow

  1. Data Preparation: We load and preprocess data from a CSV file, removing any missing values.
  2. Data Splitting: The dataset is split into training and testing sets.
  3. Feature Extraction: We use Count Vectorizer and TF-IDF Vectorizer to convert URLs into numerical features.
  4. Model Training: We train four models: Multinomial Naive Bayes (Count Vectorizer), Multinomial Naive Bayes (TFIDF), Logistic Regression (Count Vectorizer), and Logistic Regression (TFIDF).
  5. Model Evaluation: Each model is evaluated using accuracy, confusion matrix, and classification report. Results are visualized using a heatmap.
  6. Prediction: For a given URL, predictions are made using all trained models to determine if the URL is benign or malicious.

The Application

Our application is designed to analyze and classify URLs as either benign or malicious. It uses machine learning models to evaluate URL characteristics and predict their potential threat level. The application provides a user-friendly interface for submitting URLs, viewing analysis results, and accessing other features like feedback and help.
